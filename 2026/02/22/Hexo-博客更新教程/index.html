Hexo 博客更新教程

Created2026-02-22|Updated2026-02-22
|Post Views:

你想知道如何更新基于 Hexo 搭建的博客（包括新增/修改内容、同步到线上站点），对吧？Hexo 博客的更新核心是本地修改内容 → 生成静态文件 → 部署到服务器/托管平台，下面我会一步步教你完整的更新流程，新手也能轻松操作。

一、前置准备

确保你的本地环境满足以下条件：

  1. 已安装 Node.js（Hexo 运行依赖）和 Git（部署依赖）
  2. 本地有 Hexo 博客的源码目录（包含 _config.yml、source、themes 等文件夹）
  3. 已配置好 Hexo 的部署信息（比如 GitHub Pages/Gitee Pages，配置在 _config.yml 的 deploy 节点）

二、具体更新步骤

1. 进入博客本地目录

2. 常见的更新操作

（1）新增文章

这是最常用的更新场景，执行以下命令创建新文章：

执行后，Hexo 会在 source/_posts/ 目录下生成一个 Markdown 文件（比如 Hexo-博客更新教程.md），你可以用编辑器（VS Code、Typora 等）打开这个文件，编写文章内容（支持 Markdown 语法）。

（2）修改现有内容/配置
  • 修改文章：直接编辑 source/_posts/ 下对应的 Markdown 文件即可；
  • 修改博客配置（比如标题、主题、导航栏）：编辑根目录的 _config.yml 文件（注意语法：冒号后必须加空格，比如 title: 我的博客）；
  • 修改主题配置：编辑 themes/你的主题名/_config.yml 文件（比如 Next 主题的配置）。
（3）删除内容
  • 删除文章：直接删除 source/_posts/ 下对应的 Markdown 文件；
  • 删除其他页面（比如关于页）：删除 source/ 下对应的文件夹/文件（比如 source/about/index.md）。

3. 本地预览（可选但推荐）

三、常见问题解决

  1. 部署失败：检查 _config.yml 的 deploy 配置是否正确（示例如下）：
  2. 本地预览看不到更新：执行 hexo clean 后重新启动服务；
  3. 部署后内容没变化：清除浏览器缓存，或等待 CDN 刷新（部分平台有缓存）。

总结

  1. Hexo 博客更新核心流程：本地修改内容 → hexo clean → hexo g → hexo d；
  2. 新增文章用 hexo new "标题"，修改内容直接编辑对应 Markdown/配置文件；
  3. 部署前建议用 hexo s 本地预览，避免线上出错，同时记得备份源码。

按照这个流程，你可以随时更新博客的文章、配置、主题等内容，同步到线上站点。
